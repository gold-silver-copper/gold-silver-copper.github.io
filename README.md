# gold.silver.copper – Personal Website & Grift Demo

A terminal-themed personal website, blog, project showcase, and interactive REPL for the [Grift](https://github.com/skyfskyf/grift) programming language.

Built entirely with [Ratzilla](https://github.com/ratatui/ratzilla) + [TachyonFX](https://github.com/ratatui/tachyonfx) — terminal UI rendered in the browser via Rust + WebAssembly.

**Live site:** [grift.rs](https://grift.rs)

## Features

- **Interactive REPL** — Evaluate Grift expressions directly in the browser
- **Documentation** — Embedded language reference for Grift (basics, forms, advanced)
- **Blog** — Technical articles about Grift, Rust, and terminal UIs
- **Links** — Project repositories and related resources
- **Terminal aesthetic** — Fully rendered as a TUI in the browser
- **TachyonFX effects** — Procedural background animation and page transition effects
- **Clickable navigation** — All tabs, links, and buttons are clickable

### Source Code (`code/`)

All application source lives under `code/`. This is a standard Rust project built with [Trunk](https://trunkrs.dev/) (a WASM bundler for Rust).

- **`Cargo.toml`** — Declares the crate name (`grift-site`), Rust edition (2021), and all dependencies. The project is marked `publish = false` (not intended for crates.io).
- **`index.html`** — The Trunk HTML template. Trunk reads this file, injects the compiled WASM module and hashed asset references, and produces the final `index.html`. Contains:
  - Meta tags for mobile web app behavior (fullscreen, no zoom, theme color)
  - A hidden `<input>` element used to trigger the virtual keyboard on mobile for REPL input
  - Inline JavaScript for mobile interaction support (touch gestures, momentum scrolling, mousewheel, resize/orientation handling, paste support, and REPL keyboard bridging)
- **`assets/style.css`** — Global CSS for the canvas-based app. Styles the page for fullscreen canvas rendering, hides scrollbars, disables text selection, and styles the hidden REPL keyboard input.
- **`src/main.rs`** — The entire application (~1800 lines). A single-file Rust application structured as:
  - **Constants** — Static content strings for the home page description, documentation pages (basics, forms, advanced), showcase text, blog entries, and external links
  - **`Page` enum** — The six navigable pages: Home, REPL, Docs, Blog, Links, Showcase (Mobile)
  - **`ScrollTarget` enum** — Identifies which page's scroll state to update
  - **`App` struct** — All application state: current page, REPL input/history/cursor, doc/blog/scroll positions, mouse trail data, clickable area rects, TachyonFX effects, and grid dimensions
  - **Event handling** — `handle_key_event`, `handle_mouse_event`, `handle_repl_event`, `handle_blog_event`, `handle_scroll_event` methods that process keyboard and mouse input
  - **Rendering** — `draw` orchestrates the frame; dedicated `render_*` methods for each page (home, repl, docs, blog, links, showcase), plus shared helpers for background effects, tab bar, scroll arrows, and scrollable content
  - **`main()`** — Sets up the panic hook, creates the `App`, configures the WebGL2 backend with mouse selection, and starts the render loop

### Deployed Files (Root)

GitHub Pages serves the repository root. The production files at the root level are **generated** — they are the output of `build.sh` and should not be edited directly:

- **`index.html`** — The Trunk-processed HTML with hashed asset references and the WASM module loader
- **`grift-site-*.js`** — Auto-generated JavaScript bindings that load and initialize the WASM module
- **`grift-site-*_bg.wasm`** — The compiled WebAssembly binary containing the entire Rust application
- **`style-*.css`** — The CSS stylesheet with a content hash in the filename for cache busting

### Build Script (`build.sh`)

The build script automates the build-and-deploy workflow:

1. Runs `trunk build --release --public-url ./` inside `code/` to compile the Rust application to WASM
2. Removes old build artifacts (`*.js`, `*.wasm`, `style-*.css`, `index.html`) from the repository root
3. Copies the new build output from `code/dist/` to the repository root

After running `build.sh`, the updated files at the root are committed and pushed to deploy via GitHub Pages.

## Architecture

The site is a **single-page application** rendered entirely on an HTML `<canvas>` element:

1. **Rust → WASM**: The Rust application (`code/src/main.rs`) is compiled to WebAssembly by Trunk
2. **Ratzilla**: Provides the bridge between Ratatui's terminal rendering model and the browser, rendering a terminal character grid onto a WebGL2 canvas
3. **Ratatui**: The TUI framework that handles layout, widgets (paragraphs, lists, blocks, tabs), and text rendering
4. **TachyonFX**: Adds visual effects — procedural background animation, page transition effects (fade, sweep, slide, coalesce), button click effects, and tab hover glows
5. **Grift**: The embedded Lisp interpreter runs entirely in WASM, powering the interactive REPL
6. **JavaScript bridge**: Inline JS in `index.html` handles browser-specific concerns that WASM cannot directly control — touch gestures, virtual keyboard management, clipboard paste, and viewport resize events. These are forwarded to the canvas as `KeyboardEvent`s that Ratzilla processes

### Mobile Support

The inline JavaScript in `index.html` provides comprehensive mobile support:

- **Zone-aware touch handling**: The top 12% of the screen is the "tab zone" (horizontal swipes switch tabs); the rest is the "content zone" (vertical swipes scroll)
- **Momentum scrolling**: Touch velocity is tracked and decelerated for natural scroll feel
- **Virtual keyboard**: A hidden `<input>` element is focused/blurred by Rust via `js_sys::eval` calls to `window._focusReplInput()` / `window._blurReplInput()`, enabling mobile keyboard input for the REPL
- **Viewport resize**: The Visual Viewport API is used to handle virtual keyboard appearance and orientation changes
- **Paste support**: `Ctrl+V` / `Cmd+V` pastes text into the REPL by dispatching individual key events

## Building

```bash
# Install trunk (WASM bundler)
cargo install trunk

# Add the WASM target
rustup target add wasm32-unknown-unknown

# Serve locally (from the code/ directory)
cd code
trunk serve

# Build for production and copy to repo root (from the repo root)
./build.sh
```

## Navigation

| Action | How |
|--------|-----|
| Switch pages | Click the tabs at the top |
| Evaluate REPL | Type an expression and press Enter |
| Navigate docs | Click ◄ Prev / Next ► buttons or use ←/→ keys |
| Navigate blog | Click a post in the sidebar or use ↑/↓ keys |
| Open links | Click any link on the Links page |
| Scroll content | Mouse wheel, arrow keys, or swipe up/down on mobile |
| Mobile tab switching | Swipe left/right in the top tab zone |
